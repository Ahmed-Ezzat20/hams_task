# Simplified Conversation Prompts for Gemini 2.5 Flash (v2)

These prompts are optimized for use with Gemini 2.5 Flash model and structured outputs (OpenAPI schema).
The structured output ensures valid JSON is always returned.

---

## Prompt 1: Restaurant Reservation

Generate a realistic Arabic conversational dialogue between a customer and a restaurant receptionist in Saudi Arabic dialect with the following specifications:

CONTEXT:

- Customer wants to make a dinner reservation
- Restaurant is busy but has availability
- Customer has special dietary requirements

REQUIREMENTS:

1. Language: Saudi Arabic (with Modern Standard Arabic where appropriate)
2. Number of turns: 8-10 turns per speaker
3. Include natural elements:

   - Hesitations ("يعني", "خلاص", "تمام")
   - Filler words ("آه", "حسناً", "ممتاز")
   - Natural pauses
   - Some self-corrections
   - Realistic turn-taking with clear boundaries

4. Dialogue characteristics:
   - Customer starts the conversation
   - Include questions and confirmations
   - Receptionist provides information
   - Clear decision points where turns end

## Usage Notes

- Each prompt is designed to generate a realistic conversation
- Prompts are simplified for better compatibility with structured outputs
- The model will always return valid JSON matching the OpenAPI schema
- Natural elements like hesitations and filler words should be included
- Turn boundaries should be clear with proper EOU labels
- Saudi Arabic dialect should be authentic and natural
- Conversations should feel realistic and natural, not scripted

## Testing

To test these prompts:

1. Use the `gemini_eou_data_generator_v2.py` script
2. Start with 5-10 conversations to verify quality
3. Validate output with the `--validate` flag
4. Adjust temperature if needed (0.7 is recommended)
5. Generate full batches once satisfied with quality
